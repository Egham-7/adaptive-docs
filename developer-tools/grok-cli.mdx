Configure Grok CLI to use Adaptive's intelligent routing for 60-90% cost savings while maintaining the same AI-powered command line experience.

<Tip>
**Save 60-90% on AI development costs** with intelligent model routing. Works as a drop-in replacement for Grok CLI's API backend. The installer automatically configures your shell environment.
</Tip>

## Get Your Adaptive API Key

[Sign up here](https://www.llmadaptive.uk/sign-up?redirect_url=/api-platform/orgs) to create an account and generate your API key.

## Quick Setup

<Steps>
  <Step title="Run Installer" icon="download">
    ```bash
    curl -fsSL https://raw.githubusercontent.com/Egham-7/adaptive/main/scripts/installers/grok-cli.sh | bash
    ```
    The installer will automatically:
    - Install Grok CLI if not present
    - Configure Adaptive API settings
    - Add environment variables to your shell config
  </Step>
  <Step title="Verify Configuration" icon="check">
    ```bash
    grok --version
    grok "Hello, test Adaptive integration"
    ```
  </Step>
  <Step title="Start Using" icon="play">
    Begin using Grok CLI with cost-optimized intelligent routing for all your AI tasks.
  </Step>
</Steps>

```bash Environment Variables (Bash/Zsh)
echo 'export ADAPTIVE_API_KEY="your-adaptive-api-key"' >> ~/.bashrc
echo 'export ADAPTIVE_BASE_URL="https://api.llmadaptive.uk/v1"' >> ~/.bashrc
```

```fish Environment Variables (Fish Shell)
echo 'set -x ADAPTIVE_API_KEY "your-adaptive-api-key"' >> ~/.config/fish/config.fish
echo 'set -x ADAPTIVE_BASE_URL "https://api.llmadaptive.uk/v1"' >> ~/.config/fish/config.fish
```

```bash Interactive Install
curl -o grok-cli.sh https://raw.githubusercontent.com/Egham-7/adaptive/main/scripts/installers/grok-cli.sh
chmod +x grok-cli.sh
./grok-cli.sh
# Follow the interactive prompts - environment variables will be added automatically
```

## Advanced Configuration

<Tip>
The installer automatically handles environment variable setup for all major shells (bash, zsh, fish). Manual configuration is typically not needed.
</Tip>

### Custom Model Selection

```bash
# Override default intelligent routing with specific model
export ADAPTIVE_MODEL="anthropic:claude-sonnet-4-20250514"
export ADAPTIVE_MODELS='["anthropic:claude-sonnet-4-20250514","openai:gpt-4o","google:gemini-2.5-pro"]'
```

### Shell-Specific Configuration

<CodeGroup>
```bash Bash (~/.bashrc)
# Adaptive LLM API Configuration
export ADAPTIVE_API_KEY="your-adaptive-api-key"
export ADAPTIVE_BASE_URL="https://api.llmadaptive.uk/v1"
```

```zsh Zsh (~/.zshrc)
# Adaptive LLM API Configuration
export ADAPTIVE_API_KEY="your-adaptive-api-key"
export ADAPTIVE_BASE_URL="https://api.llmadaptive.uk/v1"
```

```fish Fish (~/.config/fish/config.fish)
# Adaptive LLM API Configuration
set -x ADAPTIVE_API_KEY "your-adaptive-api-key"
set -x ADAPTIVE_BASE_URL "https://api.llmadaptive.uk/v1"
```
</CodeGroup>

## Troubleshooting

<AccordionGroup>
<Accordion title="Installation Issues" icon="download">
- Ensure you have the required dependencies (Node.js, Python, etc.)
- Check installation logs for specific error messages
- Try the interactive installation method for better error handling
</Accordion>

<Accordion title="Authentication Errors" icon="key">
- Verify your API key at [llmadaptive.uk/dashboard](https://www.llmadaptive.uk/dashboard)
- Check that API key is properly set in `~/.grok/user-settings.json`
- Ensure environment variables are exported in your shell config:
  ```bash
  # Check if variables are set
  echo $ADAPTIVE_API_KEY
  echo $ADAPTIVE_BASE_URL
  
  # If not set, restart terminal or source your shell config
  source ~/.bashrc  # or ~/.zshrc for Zsh users
  ```
</Accordion>
</AccordionGroup>

## Quick Setup

<Steps>
  <Step title="Run Installer" icon="download">
    ```bash
    curl -fsSL https://raw.githubusercontent.com/Egham-7/adaptive/main/scripts/installers/codex.sh | bash
    ```
    The installer will handle configuration and environment setup automatically.
  </Step>
  <Step title="Verify Installation" icon="check">
    ```bash
    codex --version
    echo $ADAPTIVE_API_KEY
    ```
  </Step>
  <Step title="Start Coding" icon="play">
    ```bash
    codex "create a React component for user authentication"
    ```
  </Step>
</Steps>

## Alternative Setup Methods

<CodeGroup>
```bash Environment Variables
export ADAPTIVE_API_KEY='your-api-key-here'
curl -fsSL https://raw.githubusercontent.com/Egham-7/adaptive/main/scripts/installers/codex.sh | bash
```

```toml Manual Configuration
# Edit ~/.codex/config.toml
model = ""
model_provider = "adaptive"

[model_providers.adaptive]
name = "Adaptive"
base_url = "https://api.llmadaptive.uk/v1"
env_key = "ADAPTIVE_API_KEY"
wire_api = "chat"
```

```bash Interactive Install
curl -o codex.sh https://raw.githubusercontent.com/Egham-7/adaptive/main/scripts/installers/codex.sh
chmod +x codex.sh
./codex.sh
```
</CodeGroup>

## Troubleshooting

<AccordionGroup>
<Accordion title="Installation Issues" icon="download">
- Check installation method availability (npm, brew, cargo, curl)
- Verify system dependencies are installed
- Try the interactive installation method for better error handling
</Accordion>

<Accordion title="Configuration Errors" icon="gear">
- Verify API key at [llmadaptive.uk/dashboard](https://www.llmadaptive.uk/dashboard)
- Check that `ADAPTIVE_API_KEY` environment variable is set
- Restart terminal or run `source ~/.bashrc` after installation
</Accordion>
</AccordionGroup>

Adaptive uses API keys for authentication. Get your API key from the dashboard and use it with any OpenAI-compatible client or HTTP library.

## Getting Your API Key

<Steps>
  <Step title="Create Account">
    Sign up for free at [llmadaptive.uk](https://www.llmadaptive.uk) - no credit card required
  </Step>
  <Step title="Create Project">
    Create a new project or select an existing one from your dashboard
  </Step>
  <Step title="Generate API Key">
    Click "Generate API Key" and copy your key - you'll need it for requests
  </Step>
  <Step title="Secure Storage">
    Store your API key securely using environment variables or your preferred secrets manager
  </Step>
</Steps>

<Warning>
**Security**: Never expose your API key in client-side code, version control, or public repositories. Always use environment variables or secure secret management systems.
</Warning>

## Using Your API Key

### HTTP Headers

Adaptive accepts standard OpenAI-compatible authentication headers:

<Tabs>
<Tab title="Bearer Token (Recommended)">
```bash
curl https://api.llmadaptive.uk/v1/chat/completions \
  -H "Authorization: Bearer apk_123456" \
  -H "Content-Type: application/json" \
  -d '{"model": "", "messages": [{"role": "user", "content": "Hello!"}]}'
```
</Tab>

<Tab title="Alternative Format">
```bash
curl https://api.llmadaptive.uk/v1/chat/completions \
  -H "Authorization: Bearer apk_123456" \
  -H "Content-Type: application/json" \
  -d '{"model": "", "messages": [{"role": "user", "content": "Hello!"}]}'
```
</Tab>

<Tab title="Anthropic Style">
```bash
curl https://api.llmadaptive.uk/v1/messages \
  -H "Authorization: Bearer apk_123456" \
  -H "Content-Type: application/json" \
  -H "anthropic-version: 2023-06-01" \
  -d '{"model": "", "max_tokens": 1000, "messages": [{"role": "user", "content": "Hello!"}]}'
```
</Tab>
</Tabs>

### SDK Integration

<CodeGroup>
```javascript JavaScript/Node.js
import OpenAI from "openai";

const openai = new OpenAI({
  apiKey: process.env.ADAPTIVE_API_KEY,
  baseURL: "https://api.llmadaptive.uk/v1"
});

const completion = await openai.chat.completions.create({
  model: "",
  messages: [{ role: "user", content: "Hello!" }]
});
```

```python Python
import os
from openai import OpenAI

client = OpenAI(
    api_key=os.getenv("ADAPTIVE_API_KEY"),
    base_url="https://api.llmadaptive.uk/v1"
)

completion = client.chat.completions.create(
    model="",
    messages=[{"role": "user", "content": "Hello!"}]
)
```
</CodeGroup>

## Authentication Errors

### Common Error Codes

<CardGroup cols={2}>
  <Card title="401 Unauthorized" icon="x-circle">
    **Cause**: Invalid or missing API key  
    **Solution**: Check your API key and ensure it's correctly set in headers
  </Card>
  <Card title="429 Too Many Requests" icon="clock">
    **Cause**: Rate limit exceeded  
    **Solution**: Implement retry logic with exponential backoff
  </Card>
  <Card title="403 Forbidden" icon="ban">
    **Cause**: API key doesn't have required permissions  
    **Solution**: Check your subscription plan and key permissions
  </Card>
  <Card title="500 Server Error" icon="server">
    **Cause**: Temporary server issue  
    **Solution**: Retry the request after a brief delay
  </Card>
</CardGroup>

### Error Response Format

```json
{
  "error": {
    "message": "Invalid API key provided",
    "type": "invalid_request_error",
    "code": "invalid_api_key"
  }
}
```

## Security Best Practices

### API Key Management

<Steps>
  <Step title="Use Environment Variables">
    Never hardcode API keys in your source code - always use environment variables
  </Step>
  <Step title="Rotate Keys Regularly">
    Generate new API keys periodically and update your applications
  </Step>
  <Step title="Limit Key Scope">
    Use separate API keys for development, staging, and production environments
  </Step>
  <Step title="Monitor Usage">
    Regularly check your API usage and set up alerts for unusual activity
  </Step>
</Steps>

### Security Checklist

<CardGroup cols={2}>
  <Card title="✅ Do This" icon="check-circle">
    **Store keys in environment variables**  
    **Use HTTPS for all requests**  
    **Rotate keys regularly**  
    **Monitor API usage patterns**  
    **Set up usage alerts**
  </Card>
  <Card title="❌ Avoid This" icon="x-circle">
    **Hardcode keys in source code**  
    **Commit keys to version control**  
    **Share keys in chat/email**  
    **Use production keys in development**  
    **Ignore unusual usage patterns**
  </Card>
</CardGroup>


## Troubleshooting

### Common Authentication Issues

<Accordion title="API Key Not Working (401 Error)">
**Solutions**:
- Double-check your API key in the dashboard
- Ensure you're using `Authorization: Bearer apk_123456` header
- Verify environment variable is loaded correctly
</Accordion>

<Accordion title="Environment Variables Not Loading">
**Solutions**:
- Verify .env file is in project root
- Check variable names match exactly (`ADAPTIVE_API_KEY`)
- Ensure your framework loads .env files
</Accordion>
